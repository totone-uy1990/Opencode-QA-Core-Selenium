# Contexto del Proyecto de Automatización

- **Usuario**: Ronal, QA Automation Engineer.
- **Stack Tecnológico**: Java 24, Gradle, Selenium WebDriver, Cucumber, TestNG/JUnit 5 y Allure Reports.
- **Arquitectura**: Page Object Model (POM) estricto con separación de capas.
- **Idioma**: Explicaciones y comentarios siempre en español.

## Directrices de Trabajo
1. **Prioridad de Skills**: Cuando trabaje en archivos `.feature`, activa `cucumber-bdd-expert`. Cuando trabaje en `src/main/java`, activa `automation-framework-architect` y `qa-automation-senior`.
2. **Ejecución**: Usa siempre Gradle a través de `automation-ops-manager` para cualquier tarea de build o test.
3. **Estabilidad**: Antes de sugerir cualquier cambio en un Page Object, utiliza el **Browser Agent** para verificar la robustez de los selectores siguiendo las reglas de `selenium-stability-specialist`.
4. **Seguridad**: Nunca sugieras cambios que rompan el paralelismo basado en `ThreadLocal`.
